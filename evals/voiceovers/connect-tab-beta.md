# connect-tab-beta — Desktop Connect tab beta shell

Cast is Alex, the Acme Robotics admin, using the OpenWork desktop app against the local Den stack. The proof stays cloud-only: Connect shows organization MCP connections, while Extensions keeps the old local extension experience.

1. Alex signs the desktop into Acme and opens Settings. In the Cloud group, there is a new Connect entry, and it carries a small Beta badge so the rollout is visible before anyone opens the tab.

2. With Acme's Connect capability turned off and no usable organization connections, the Connect tab does not look broken. It shows the friendly pitch, tells Alex to ask an org admin to enable Connect beta, and links admins back to Den web.

3. After the platform admin enables Acme's capability and Alex publishes a test per-member MCP connection, the same tab turns active. It shows the organization status row and the beta connection card with Connect your account, without any local quick-connect content.

4. Alex switches back to Extensions. The familiar My Extensions sections and quick-connect language are still there, but the organization connection is gone from both My Extensions and Marketplace — Connect is now the only desktop home for those cards.
