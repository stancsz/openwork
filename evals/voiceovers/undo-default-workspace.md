# undo-default-workspace — Restore user-controlled desktop onboarding

1. On desktop, whenever no workspaces exist—including a fresh launch—OpenWork shows the welcome screen instead of silently creating one.

2. The user chooses a folder, and OpenWork creates the workspace only at that location.

3. The existing provider and onboarding steps appear before entering the workspace.

4. OpenWork opens the newly created session, ready for the first message.

5. Opening an empty existing workspace does not automatically create a session; the user remains in control.
