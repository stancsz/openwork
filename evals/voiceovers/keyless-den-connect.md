# keyless-den-connect — A Den user downloads the normal app and connects without deployment keys

This flow proves the default organization download experience requires no signing-key or customer-specific installer configuration.

1. I open my organization’s desktop setup page and see one simple path: download the normal OpenWork app, open it for my organization, and sign in.

2. I click Download and the standard installer starts immediately; Den does not make me wait for a custom ZIP or a customer-specific build.

3. After installation I return to the same page and click Open OpenWork, where the app shows the exact organization and Den server before changing anything.

4. I confirm the connection and arrive at the normal organization sign-in, with my organization’s name and wordmark already in place.

5. If I reuse or alter the short-lived connection link, OpenWork refuses it clearly and leaves the existing desktop configuration untouched.

6. An operator can later enable signed handoffs without changing this download experience or producing a different installer.
