# connector-health-recovery — Name the connector that needs attention

1. I ask OpenWork Cloud to use one of my organization's connected services, and capability discovery checks that connector live.

2. When that connector's credential is invalid, OpenWork Cloud says the named downstream connector needs reconnection, identifies whether I or an administrator must act, and keeps the healthy OpenWork Cloud connection distinct.

3. After the connector is fixed, I retry discovery in the same task and continue without reconnecting OpenWork Cloud.
