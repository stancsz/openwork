# desktop-notifications — Preferences control native desktop notifications

The demo shows a user choosing which OpenWork events can become native operating
system notifications from the desktop app.

1. In Preferences, I open the new Desktop Notifications setting and see three choices: Off, Important, and All.

2. I choose Important, so OpenWork will only send native OS notifications for events that need attention, like errors, questions, or permission requests.

3. When a session needs attention, OpenWork sends a native desktop notification whose text says what happened, instead of a generic “Session completed.”

4. I switch the setting to All, run a task, and when the session finishes, the desktop notification clearly says the task completed.
